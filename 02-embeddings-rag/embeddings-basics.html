What Are Embeddings? ü§î

üéØ Simple Definition - Like Raju's Strategic Planning!

Embeddings are numerical representations of text that capture meaning. Think of them as coordinates in a high-dimensional space where similar texts are close to each other - just like how Raju maps out strategies for Dholakpur!

Imagine you could plot every word, sentence, or document on a map where:

  • Similar meanings cluster together
  • Distance represents semantic similarity
  • You can mathematically calculate how "close" two texts are

Types of Embeddings

1. Text Embeddings

  • Word embeddings: Individual words ‚Üí vectors
  • Sentence embeddings: Complete sentences ‚Üí vectors
  • Document embeddings: Entire documents ‚Üí vectors

2. Other Types

  • Image embeddings: Pictures ‚Üí vectors
  • Audio embeddings: Sounds/music ‚Üí vectors

Basic Embedding Example

import numpy as np
from sentence_transformers import SentenceTransformer

# Load the embedding model locally
model = SentenceTransformer("all-mpnet-base-v2")

def get_embedding(text):
    """Get embedding for a piece of text."""
    # Encode the text and convert to list
    return model.encode(text).tolist()

# Test it out
text = "I love programming in Python"
embedding = get_embedding(text)

print(f"Text: {text}")
print(f"Embedding dimension: {len(embedding)}")
print(f"First 5 values: {embedding[:5]}")

Calculating Similarity

Once you have embeddings, you can calculate how similar two pieces of text are using cosine similarity:

import numpy as np

def cosine_similarity(vec1, vec2):
    """Calculate cosine similarity between two vectors."""
    # Convert to numpy arrays
    vec1 = np.array(vec1)
    vec2 = np.array(vec2)
    
    # Calculate cosine similarity
    dot_product = np.dot(vec1, vec2)
    norm1 = np.linalg.norm(vec1)
    norm2 = np.linalg.norm(vec2)
    
    similarity = dot_product / (norm1 * norm2)
    return similarity

# Example: Compare similar and different texts
text1 = "I love programming in Python"
text2 = "Python programming is my passion"
text3 = "I enjoy cooking Italian food"

# Get embeddings
embedding1 = get_embedding(text1)
embedding2 = get_embedding(text2)
embedding3 = get_embedding(text3)

# Calculate similarities
sim_1_2 = cosine_similarity(embedding1, embedding2)
sim_1_3 = cosine_similarity(embedding1, embedding3)

print(f"Similarity between text1 and text2: {sim_1_2:.3f}")
print(f"Similarity between text1 and text3: {sim_1_3:.3f}")
‚úÖ Expected Output 
Similarity between text1 and text2: 0.850  ‚Üê High similarity!
Similarity between text1 and text3: 0.360  ‚Üê Low similarity
üìä Similarity Scores
  • 1.0: Identical or nearly identical meaning
  • 0.8-0.9: Very similar concepts
  • 0.6-0.7: Somewhat related
  • 0.3-0.5: Loosely related
  • 0.0-0.2: Very different concepts

Building a Document Similarity Finder

Let's build a practical application that finds the most similar documents:

import numpy as np
from sentence_transformers import SentenceTransformer

class DocumentSimilarityFinder:
    def __init__(self):
        self.documents = []
        self.embeddings = []
        # Load the embedding model locally
        self.model = SentenceTransformer("all-mpnet-base-v2")
    
    def get_embedding(self, text):
        """Get embedding for text."""
        return self.model.encode(text).tolist()
    
    def add_document(self, document):
        """Add a document to the collection."""
        embedding = self.get_embedding(document)
        self.documents.append(document)
        self.embeddings.append(embedding)
        print(f"Added document: {document[:50]}...")
    
    def cosine_similarity(self, vec1, vec2):
        """Calculate cosine similarity."""
        vec1 = np.array(vec1)
        vec2 = np.array(vec2)
        return np.dot(vec1, vec2) / (np.linalg.norm(vec1) * np.linalg.norm(vec2))
    
    def find_similar(self, query, top_k=3):
        """Find most similar documents to query."""
        if not self.documents:
            return []
        
        query_embedding = self.get_embedding(query)
        similarities = []
        
        for i, doc_embedding in enumerate(self.embeddings):
            similarity = self.cosine_similarity(query_embedding, doc_embedding)
            similarities.append((similarity, i, self.documents[i]))
        
        # Sort by similarity (highest first)
        similarities.sort(reverse=True)
        return similarities[:top_k]

# Demo usage
finder = DocumentSimilarityFinder()

# Add some documents
documents = [
    "Python is a great programming language for data science and machine learning.",
    "I love cooking pasta with fresh tomatoes and basil.",
    "Machine learning algorithms can predict future trends from historical data.",
    "The best pizza has a crispy crust and fresh mozzarella cheese.",
    "Data scientists use Python and R for statistical analysis.",
    "Italian cuisine is known for its use of fresh herbs and quality ingredients."
]

for doc in documents:
    finder.add_document(doc)

# Search for similar documents
print("\n" + "="*60)
query = "What programming language is good for AI?"
print(f"Query: {query}")
print("\nMost similar documents:")

results = finder.find_similar(query, top_k=3)
for i, (similarity, idx, document) in enumerate(results, 1):
    print(f"{i}. Similarity: {similarity:.3f}")
    print(f"   Document: {document}")
    print()
