Understanding the AI Connection üîå

Snape observes: "Even a first-year should comprehend this. It's merely sending instructions and awaiting a response. Do try not to complicate it unnecessarily."

The OpenAI API is like sending an owl to a very smart wizard. We ask a question, it thinks, and sends back a helpful answer. Let's learn how to do this properly!

ü™Ñ The Simple 4-Step Process (Works for both OpenAI & Gemini!)
  1. Import: Load the OpenAI or Gemini tools and your API key
  2. Initialize: Create a connection to the AI service
  3. Send: Make your API call with your question
  4. Receive: Get and display the AI's response

Your First Magical API Call! ü™Ñ

Choose your path: OpenAI (paid, powerful) or Gemini (free, great for learning!)

ü§ñ OpenAI (ChatGPT)

Paid: ~$0.002 per 1K tokens
Model: gpt-3.5-turbo

1. Create first_openai.py:

import os
from dotenv import load_dotenv
from openai import OpenAI

# Load environment variables
load_dotenv()

# Initialize OpenAI client
client = OpenAI(
    api_key=os.getenv("OPENAI_API_KEY")
)

# Make API call
response = client.chat.completions.create(
    model="gpt-3.5-turbo",
    messages=[{
        "role": "user",
        "content": "Say hello!"
    }]
)

# Print response
print("ü™Ñ AI Response:")
print(response.choices[0].message.content)

2. Run it:

python first_openai.py
‚úÖ Output:

"Hello! I'm ChatGPT, an AI assistant..."

2Ô∏è‚É£ Temperature Control üå°Ô∏è

Temperature controls creativity: 0 = focused, 2 = creative

Setting Temperature (0-2):

from openai import OpenAI
import os

Understanding the Code üìñ

Part 1: Setting Up

from openai import OpenAI
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

Ron explains: "This creates our connection to AI, like having your wand at the ready!"

Part 2: The API Call

response = client.chat.completions.create(
    model="gpt-3.5-turbo",
    messages=[
        {"role": "user", "content": "Your question here"}
    ]
)
ü™Ñ Parameters Explained
  • model: Which AI brain to use (gpt-3.5-turbo is fast & cheap!)
  • messages: Your conversation messages
  • role: Can be "user" (you), "assistant" (AI), or "system" (instructions)
  • content: Your actual message text

Part 3: Getting the Response

print(response.choices[0].message.content)

Ron explains: "This extracts the AI's answer, like reading the reply from your owl!"

Error Handling üõ°Ô∏è

Snape warns: "Error handling is not optional. Unlike certain... reckless students... a proper programmer anticipates failure and prepares accordingly."

import os
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

try:
    response = client.chat.completions.create(
        model="gpt-3.5-turbo",
        messages=[
            {"role": "user", "content": "Hello from Hogwarts!"}
        ]
    )
    print("‚úÖ Success!")
    print(response.choices[0].message.content)
    
except Exception as e:
    print(f"‚ùå Error: {e}")
    print("üí° Ron suggests: Check your API key!")
‚ö†Ô∏è Common Problems
  • AuthenticationError: Wrong API key
  • RateLimitError: Too many requests (wait a bit!)
  • APIError: OpenAI service issue

Tracking Costs üí∞

See how much each request costs:

response = client.chat.completions.create(
    model="gpt-3.5-turbo",
    messages=[
        {"role": "user", "content": "Hello!"}
    ]
)

# Check token usage
usage = response.usage
print(f"üìä Tokens:")
print(f"   Question: {usage.prompt_tokens}")
print(f"   Answer: {usage.completion_tokens}")
print(f"   Total: {usage.total_tokens}")

# Calculate cost
cost = (usage.total_tokens / 1000) * 0.002
print(f"üí∞ Cost: ${cost:.6f}")
